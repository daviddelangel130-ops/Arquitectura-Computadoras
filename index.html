Arquitectura de Computadoras

Portal académico sobre la estructura, funcionamiento y diseño de sistemas computacionales.

Unidad 1: Modelos de Arquitecturas de Cómputo

1.1 Modelos de Arquitectura

La arquitectura de computadoras define la base conceptual y técnica para la creación de sistemas funcionales, gestionando cómo los componentes se comunican y ejecutan instrucciones.

Clásico (Von Neumann)

Usa una sola memoria para instrucciones y datos. Es eficiente pero genera un cuello de botella al compartir el mismo bus.

Segmentadas (Harvard)

Utiliza memorias separadas para instrucciones y datos, permitiendo que el CPU trabaje con ambas al mismo tiempo.

1.2 Análisis de los Componentes

El sistema se divide en tres áreas críticas de interacción:

1.2.1 Hardware

Componentes físicos como CPU, Motherboard y Buses.

1.2.2 Software

Sistemas operativos y aplicaciones que controlan el flujo lógico.

1.2.3 Periféricos

Dispositivos de Entrada/Salida que conectan al usuario con la máquina.

1.3 Vistas de la Arquitectura

Existen diferentes niveles de abstracción según el rol:

  • Programador: Ve el conjunto de instrucciones (ISA) y registros.
  • Diseñador: Ve buses, lógica digital y microarquitectura.
  • Sistemas: Ve la integración total y jerarquías de memoria.

1.4 Clasificación de Flynn

Clasifica las máquinas según el flujo de instrucciones y datos:

SISD y SIMD

Sistemas monoprocesador tradicionales (SISD) vs Procesamiento de vectores como GPUs (SIMD).

MISD y MIMD

Múltiples instrucciones para un dato (MISD) vs Sistemas multinúcleo modernos (MIMD).

1.5 Jerarquía de Buses

Define cómo se transporta la información. Se utilizan buses de Datos (valor), Direcciones (ubicación) y Control (señales de sincronización).

Unidad 2: Estructura y Funcionamiento del Procesador

2.1 Organización del Procesador

El procesador o CPU es el cerebro del sistema. Su función es interpretar y ejecutar instrucciones coordinando todas las operaciones internas. Se basa en tres funciones principales:

Control Central

Actúa como director de orquesta, coordinando las señales para que los componentes trabajen en orden.

Procesamiento

Ejecuta cálculos aritméticos y lógicos sobre los datos recibidos.

Pipelining

Técnica de segmentación que permite acelerar la ejecución al procesar varias instrucciones en diferentes etapas simultáneamente.

2.2 Estructura de Registros

Los registros son memorias de ultra alta velocidad dentro del procesador. Se clasifican principalmente en dos tipos:

Visibles para el Usuario

Permiten al programador de lenguaje máquina o ensamblador minimizar referencias a la memoria principal. Incluyen registros de datos, direcciones e índices.

Control y Estado

Utilizados por la unidad de control para supervisar la ejecución. Los más importantes son:

  • Program Counter (PC): Dirección de la próxima instrucción.
  • Instruction Register (IR): La instrucción que se está ejecutando.
  • Status Register (Flags): Indica resultados de operaciones (Cero, Acarreo, Signo).

2.3 El Ciclo de Instrucción

Es el proceso que sigue el procesador para ejecutar cada instrucción. Según tu material, se divide en las siguientes fases críticas:

  1. Fetch (Búsqueda): Se trae la instrucción desde la memoria al procesador.
  2. Decode (Decodificación): Se interpreta qué es lo que la instrucción debe hacer.
  3. Execute (Ejecución): Se realiza la operación (suma, movimiento de datos, etc.).
  4. Writeback (Escritura): Se guardan los resultados en la memoria o registros.

2.4 Modos de Direccionamiento

Definen cómo se calculan las direcciones de memoria de los operandos. Algunos ejemplos clave son el Direccionamiento Directo, Indirecto e Indexado (muy usado en sistemas ARM para control de periféricos).

Unidad 3: Selección de Componentes para Ensamble

3.1 El Chipset (Centro de Control)

Es el conjunto de circuitos que coordina el flujo de datos entre el CPU y el resto de los componentes. Su elección determina qué tan actualizable será tu equipo y qué tecnologías (como PCIe 4.0 o 5.0) podrá soportar.

3.2 Criterios de Selección de Hardware

Para un ensamble profesional, se deben analizar los siguientes puntos técnicos del documento:

Unidad Central de Procesamiento

Se debe considerar la arquitectura (núcleos/hilos), la frecuencia de reloj y la compatibilidad con el socket de la tarjeta madre.

Módulos de Entrada y Salida

Selección de periféricos según la tasa de transferencia y el tipo de conexión (USB-C, Thunderbolt, DisplayPort) para evitar latencias.

Dispositivos de Almacenamiento

Comparativa entre HDD (almacenamiento masivo lento) y SSD/NVMe (arranques rápidos y carga de software inmediata).

Fuentes de Alimentación

No solo es el wattage; es vital revisar la eficiencia energética (Certificación 80 Plus) para proteger los componentes internos.

3.3 Aplicaciones en Ambientes de Servicio

La arquitectura se adapta a las necesidades del entorno:

Negocios y Comercio:

Automatización de bases de datos, gestión de inventarios y seguridad en transacciones digitales.

Industria y Manufactura:

Sistemas embebidos para control de robótica, sensores de calidad y logística automatizada.

Unidad 4: Memorias

4.1 Memoria RAM (Acceso Aleatorio)

La RAM es un componente volátil que almacena datos de forma temporal para que el procesador acceda a ellos rápidamente. Su función principal es mejorar el rendimiento del sistema permitiendo ejecutar aplicaciones de manera eficiente.

Almacenamiento Temporal

Carga los datos necesarios de los programas abiertos. Al ser volátil, la información se borra al apagar el equipo.

Multitarea

Una mayor cantidad de RAM permite que la computadora funcione de manera fluida con múltiples aplicaciones simultáneas.

4.2 Tipos de Memoria y Características

La evolución de la memoria se mide por su velocidad (frecuencia) y latencia:

Tecnología DDR

Desde DDR3 hasta la actual DDR5, cada generación aumenta la tasa de transferencia y reduce el consumo de energía.

Frecuencia (MHz)

Determina la velocidad de comunicación. Equipos modernos operan desde 3200MHz hasta más de 6000MHz en gamas altas.

Latencia CAS (CL)

Es el tiempo que tarda la memoria en responder a una solicitud. Valores más bajos indican un rendimiento superior.

4.3 Clasificación y Selección por Gamas

Según el análisis de componentes del material, así se seleccionan las memorias según el uso:

  • Gama Media: 8 GB a 16 GB de RAM. Ideal para software de oficina, navegación fluida y diseño básico.
  • Gama Alta (Premium): 32 GB a 64 GB de RAM. Utiliza frecuencias de 6000 MHz en adelante y latencias bajas (CL30). Diseñada para usuarios que buscan potencia extrema, videojuegos de última generación y renderizado profesional.

4.4 Compatibilidad con Chipsets

La memoria debe ser compatible con la placa base. Para Intel se utilizan series como la Z790 y para AMD las series X670 en configuraciones de alto rendimiento.

Unidad 5: Procesamiento Paralelo

5.1 Fundamentos del Paralelismo

La computación paralela es una forma de cómputo en la que muchas instrucciones se ejecutan simultáneamente. Se basa en el principio de que los problemas grandes se pueden dividir en partes pequeñas para resolverse al mismo tiempo.

División del Problema

Fragmentación de tareas para que cada núcleo del procesador trabaje en una parte específica.

Sincronización

Coordinación entre procesadores para asegurar que los datos lleguen en el orden correcto.

Escalabilidad

Capacidad del sistema para mejorar su rendimiento al añadir más recursos de procesamiento.

5.2 Clasificación y Tipos

El paralelismo se puede aplicar en diferentes niveles del sistema:

Paralelismo a nivel de Instrucción

Uso de Pipelining para que el CPU empiece a procesar la siguiente instrucción antes de terminar la actual.

Paralelismo de Datos

Una misma operación se aplica a un conjunto masivo de datos (común en procesamiento de video).

5.3 Organización y Topologías de Red

Define cómo se conectan los nodos para intercambiar información:

  • Malla (Mesh): Los nodos se conectan a sus vecinos, ideal para servidores masivos.
  • Anillo (Ring): Conexión circular, usada históricamente en buses internos de CPUs de escritorio.
  • Estrella: Todos los nodos se conectan a un conmutador central.

5.4 Arquitecturas de Vanguardia

Análisis de las soluciones actuales en el mercado:

AMD: Infinity Fabric

Uso de Chiplets (pequeños trozos de silicio) conectados por una red ultra rápida para mejorar la eficiencia.

NVIDIA: GPU Computing

Miles de núcleos pequeños trabajando en paralelo para Inteligencia Artificial y renderizado gráfico.

Intel: Mesh/Ring Bus

Optimización de la comunicación interna entre núcleos de alto rendimiento y núcleos de eficiencia.
